# Documentation Naming Convention

## Research and Plan Documents

All research and plan documents should be stored in the `/docs` folder with unique names to avoid overwriting files.

### Naming Pattern
- Use descriptive, unique names
- Include date or version if needed for tracking
- Avoid generic names like "plan.md" or "research.md"
- Use hyphens or underscores for multi-word names

### Examples
- `wfc-phase1-plan.md` ✓
- `wfc-research-structure.md` ✓
- `wfc-algorithm-research-2026-06-18.md` ✓
- `castle-generation-design.md` ✓

### Rules
- Never overwrite existing files
- Always check if a file exists before creating
- Use unique identifiers (dates, versions, prefixes)
