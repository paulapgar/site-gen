# Documentation Naming Convention

## Research and Plan Documents

All research and plan documents should be stored in the `/docs/research` folder with unique names to avoid overwriting files.

## Coding examples and prefered methods using Excalibur

All the coding example files are in `/docs/coding` folder and should be referenced whenever
areas of the Excalibur engine are used.

## Design documents

All design documents are in `/docs/design`

## Rules
- Never overwrite existing files
- Always check if a file exists before creating
- Use unique identifiers (dates, versions, prefixes)

### Naming Pattern
- Use descriptive, unique names
- Include date or version if needed for tracking
- Avoid generic names like "plan.md" or "research.md"
- Use hyphens or underscores for multi-word names
- Can use format `<topic>-<doc_type>-<subtype>` for example `wfc-plan-phase1`

### Examples
- `wfc-plan-phase1.md` ✓
- `wfc-research-structure.md` ✓
- `wfc-research-algorithm-2026-06-18.md` ✓
- `castle-design-generation.md` ✓
